정렬 기능 위해 신경 쓴 부분은 '2가지'
1) sequelize 활용해서 필요한 데이터를, 어떻게 가져올 것 인가. 
2) 가져온 데이터를 html 에서 '그려주는' 작업 

문제는 2번째 부분에서 발생. 

'그려주는 기능' 은 'getView 함수' 에서 담당. 

getview 함수는 1) 데이터를 가져와 '배열' 에 담고 2) 배열에 대해 for 문을 돌리면서, 필요한 태그 및 text 를 넣는 기능을 한다. 

이때 기존 게시물이 누적되지 않게 하기 위해, 매번 그릴 때 마다, '초기화 하는 기능' 을 넣었음. 

db 가져와서 -> 가져온대로 그리고 -> 그 다음에 다시 초기화 되는 문제가 발생. | 다른 텍스트 데이터, | 






data 를 가져와서, 



새로 가져온 정렬 데이터가 실행되고, 그 다음 다시 getview 함수가 실행되어, 다시 원래 초기의 view 로 돌아가는 문제 

getview 함수의 초기화 기능을 없애면, 새로고침 할 때, 게시물이 많이 생겨 없앨 수 없음. 

so, getview함수 기능을 유지하면서, 새로 받아온 기능을 html 에 렌더해야 했고, 이를 위해, 완전히 다른 페이지로 연결 시킴.
