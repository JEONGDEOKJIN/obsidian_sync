

- API 란? 
``` bash 
특정 프로그램이 소유하고 있는 '기능', '자원', '데이터' 에, 
다른 프로그램이 접근할 수 있도록 만든것을
'interface' 이라 함. 

so, interface 를 통해서, 해당 프로그램이 가진 리소스를 사용할 수 있다. 

API 는 Application Programming Interface 를 의미함. 
즉, '어플리케이션' 간 상호작용할 수 있도록 만든 '인터페이스' 임. 
```

- Web API 란? 
``` BASH 
웹 어플리케이션 개발을 위해, '브라우저' 가 제공하는 API. 

즉, 예를 들어, 웹 어플리케이션 개발에 필요한 비동기 처리 함수인 XMLHttpRequest, fetch 등의 기능을 브라우저가 미리 만들어두고 -> API 를 통해서 -> 다른 어플리케이션에서도 사용할 수 있게 하는 것. 
```

- RESTful API (REST API)
``` BASH
- 'REST 원칙'을 기반으로, 구현한 API

- 클라이언트가, 서버가 갖고 있는 리소스, 데이터, 등을 사용하기 위해서, REST 원칙을 지켜서 만든 API 
ex) 

- REST API 는 '자원', '행위', '표현' 의 3요소로 구성된다. 
	- 자원 : URI (엔드포인트), 명사, 
	- 행위 : 자원에 대한 행위, HTTP 요청 메서드, 동사 
	- 표현 : 자원에 대한 행위의 구체적인 내용, 페이로드, 
```


- REST API 설계 원칙 
``` BASH 
1. URI 는 '리소스' 를 '명사' 로 표현 

# bad 
GET /getTodos/1       # 'Todos 에 대한 리소스를 get 한다.' 라고 표현하지 않는다. 
GET /todos/show/1   # 동사형으로 끝나면, 해당 리소스가 제대로 표현되지 않는다. 

# good
GET /todos/1     # todos 리소스를 명사형으로 표현


2. '행위' 는 'HTTP 요청 메서드' 를 통해서! 
	- 리소스에 대한 행위는 URI 에 표현하지 않는다. 
# bad
GET /todos/delete/1      # 리소스를 동사로 표현했기 때문에 bad 

# good 
DELETE /todos/1     # 삭제 행위를 HTTP 요청 메소드로 표현! 

```
