

# API 란? 
``` bash 
API 는 Application Programming Interface 를 의미.
'어플리케이션' 간 상호작용할 수 있도록 만든 '인터페이스' 임. 

'인터페이스' 란, A 시스템과 B 시스템이 상호작용(커뮤니케이션) 할 수 있게 하는 '지점(매개체, 방법)' 정도
예를 들어, A 시스템에 특정 자원이 있는 경우, B 시스템에서는 '매개체' 를 통해 A 시스템 안에 있는 자원을 사용할 수 있음. 

컴퓨터 사이언스에서, 인터페이스는, 
특정 프로그램이 소유하고 있는 '기능', '자원', '데이터' 에, 
다른 프로그램이 접근할 수 있도록 정의한 
규칙, 규격, 표준 임. 

쉽게 말하면, 
원하는 걸 받으려면, 
정해진 '약속(추상적인 매개체)' 대로 해야 함.  

다시 쉽게 말하면, ⭐⭐⭐ 
특정 어플리케이션(시스템)이 갖고 있는 자원 등을 
다른 시스템에서도 사용할 수 있게 하는 '매개체' 임. 
이 매개체를, 프로그래밍을 통해 만든 것. 


CF. Web API 란? 
- '브라우저' 가 '웹 개발'에 필요한 기능을 사전이 만들어둠 
	- ex) 비동기 HTTP 통신 = XMLHttpsRequest, fetch 등 
- 외부(ex. js 엔진) 에서, 규칙(추상적 매개체)을 준수하면 -> 해당 기능을 사용할 수 있게 함
```

# REST 원칙 
``` BASH
- 문제 상황 
	- 백엔드 환경에 데이터가 저장되어 있음. 
	- 프론트에서 해당 데이터가 필요함. ex) 현재 유저의 id, 좋아요 개수 등 
	- URL 을 적어야 하는데, 어떤 규칙하에 적으면, 혼란이 적을까? 


- REST 이란 
	- HTTP 통신 에서, '클라이언트가 서버의 리소스에 접근'하는 '방식'을 규정한 '⭐아키텍처' ⭐⭐⭐ #📛📛📛

- REST 구성 요소
	- 필요한 리소스를 요청할 때, '자원', '행위', '표현' 의 3요소로 요청한다. 
	- 자원 : 필요한 리소스. URI(엔드포인트), 명사
	- 행위 : 자원에 대한 행위, HTTP 요청 메서드, 동사 
	- 표현 : "이 자원에 대해 필요해요~ 라고 표현", 페이로드

(출처 : 모던 자바스크립트 p830)
```

# RESTful API (REST API)
``` BASH
- 생각해 볼 것 
	- API 가 '시스템과 시스템 간 자원 교류' 의 관점이었음 -> 그러면, REST API 는 어떤 시스템간의 교류? 

- 의의
	- 'REST 원칙'을 기반으로, '서비스 API' 를 구현 (출처 : 모던 자바스크립트 p830) ⭐⭐⭐ 

- 특징
	- '자체 표현 구조' 로써, REST API 만으로, 'HTTP 요청을 이해' 할 수 있음 . (출처 : 모던 자바스크립트 P830)⭐⭐⭐ 
	- 어떤 리소스를 요구하는지, 해당 리소스로 무엇을 할 것 인지, 그 행위를 할 때 필요한 추가 정보는 무엇인지! 까지를 모두 알 수 있음. 

. (출처 : 모던 자바스크립트 P830)
```


- REST API 설계 원칙 
``` BASH 
1. URI 는 '리소스' 를 '명사' 로 표현 

# bad 
GET /getTodos/1       # 'Todos 에 대한 리소스를 get 한다.' 라고 표현하지 않는다. 
GET /todos/show/1   # 동사형으로 끝나면, 해당 리소스가 제대로 표현되지 않는다. 

# good
GET /todos/1     # todos 리소스를 명사형으로 표현


2. '행위' 는 'HTTP 요청 메서드' 를 통해서! 
	- 리소스에 대한 행위는 URI 에 표현하지 않는다. 
# bad
GET /todos/delete/1      # 리소스를 동사로 표현했기 때문에 bad 

# good 
DELETE /todos/1     # 삭제 행위를 HTTP 요청 메소드로 표현! 
```

# HTTP 요청 유형
``` BASH
- GET 요청 
	- 특정 리소스를 '취득(retrieve)' 함 

- POST 
	- 페이로드의 MIME 타입을 지정 ⭐⭐ 

- 기타 
	- PUT, PATCH, DELETE 등 

- 시사점
	- SPA 를 활용할 때, HTTP 요청을 반드시 하게 된다. 
	- 새로운 스택을 공부할 때, REST API 기반으로 정리할 수 있을 것 이다. 
```
